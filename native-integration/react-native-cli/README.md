# Applaud IQ — React Native (bare CLI) embed example

A **bare React Native (CLI)** example showing **manual + auto login and SSO** with the
[`@applaudiq/embed-react-native`](https://www.npmjs.com/package/@applaudiq/embed-react-native) SDK.

> Prefer **Expo**? See [`../react-native-expo/`](../react-native-expo/) — the SDK + integration code are
> identical; only scheme registration differs (Expo does it with one line in `app.json`).

This folder ships the **integration sources** (`App.tsx`, `config.ts`, `MintClient.ts`, `index.js`,
`metro.config.js`). A bare RN app's `ios/` + `android/` native projects are generated by the RN CLI, so you
scaffold the host once and drop these in — then register the SSO scheme in two native files (step 2).

## What you'll see

- **Manual login** — the portal's own email/SSO login inside the embed. No server, no token.
- **Auto-login** — your backend mints a one-time token (`MintClient.ts`); the embed signs in silently.
- **SSO** — the SDK opens the system browser; the result returns on `aiqrncli://sso-callback`. On success the
  code is exchanged in the WebView; on failure it fires `onError` and reloads the login.

## 1. Scaffold + install

```sh
# create the host app (RN 0.74), then copy the files from this folder over the generated ones:
npx @react-native-community/cli init AiqRnCliExample --version 0.74.5
#   → copy App.tsx, config.ts, MintClient.ts, index.js, metro.config.js, package.json into it
npm install                     # installs @applaudiq/embed-react-native (npm) + react-native-webview
cd ios && pod install && cd ..  # iOS only
```

> **App icon (optional):** the host ships React Native's default icon. To use your own, open
> `android/.../AndroidManifest`'s `@mipmap/ic_launcher` in **Android Studio → Asset Studio** (or replace the
> `mipmap-*/ic_launcher*.png` files + the `mipmap-anydpi-v26/ic_launcher.xml` adaptive icon), and on iOS set the
> `AppIcon` set in `ios/.../Images.xcassets`. (Expo does this from one `app.json` field — see the Expo example.)

## 2. Register your SSO callback scheme (the key step)

Pick a scheme **unique to your app** and register it natively. It must match `Config.SSO_CALLBACK`
(`aiqrncli://sso-callback`). The SDK sends it to the backend as `native_redirect`, so the callback returns to
exactly this app — no Android "Open with" chooser when two Applaud IQ apps are installed.

**iOS** — `ios/AiqRnCliExample/Info.plist`:

```xml
<key>CFBundleURLTypes</key>
<array>
  <dict>
    <key>CFBundleURLSchemes</key>
    <array><string>aiqrncli</string></array>
  </dict>
</array>
```

**Android** — `android/app/src/main/AndroidManifest.xml`, inside the main `<activity>`:

```xml
<intent-filter>
  <action android:name="android.intent.action.VIEW" />
  <category android:name="android.intent.category.DEFAULT" />
  <category android:name="android.intent.category.BROWSABLE" />
  <data android:scheme="aiqrncli" android:host="sso-callback" />
</intent-filter>
```

## 3. Configure + run

1. Edit `config.ts` — your `PUBLISHABLE_KEY` and `BASE_URL`. From an emulator the local portal is
   `http://10.0.2.2:3017` (Android) / `http://localhost:3017` (iOS sim). Auto-login: point `MINT_ENDPOINT` at your
   mint server, or set `TEST_EMBED_SECRET` for a quick local test (never ship a real secret).
2. `npx react-native run-android` or `npx react-native run-ios`.
3. Tap **Manual login** (works immediately) or **Auto-login**, then **Continue with Google/Microsoft** for SSO.

## How it consumes the SDK

`package.json` installs the published SDK from npm:

```json
"@applaudiq/embed-react-native": "^1.1.1"
```

`metro.config.js` is the plain React Native default — the SDK resolves from `node_modules` like any other dep.

## Troubleshooting

- **SSO returns but nothing happens** — the native scheme (step 2) must equal `Config.SSO_CALLBACK`, on the
  platform you're running.
- **Metro can't resolve the SDK** — re-run `npm install`, then restart Metro with a clean cache
  (`npx react-native start --reset-cache`).
